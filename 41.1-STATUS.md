# Shen 41.1 Port Status for shen-lua

This document describes the current state of the Shen 41.1 port (targeting `ShenOSKernel-41.1/klambda`).

## What Works

- The full 41.1 KLambda kernel loads successfully:
  - All 21 `.kl` files (core + `stlib`, `compiler`, `extension-*`, etc.).
  - `shen.initialise` completes without fatal error.
  - `version` reports `"41.1"`.
- Basic primitives function (`+`, `cons`, `load`, `eval-kl`, etc.).
- Simple KLambda evaluation works via `run_kl_string` / `eval`.
- Platform globals are set (`*language*`, `*implementation*`, `*port*`, `*porters*`, `*os*`, `*release*`).
- `*macros*`, `shen.*tc*`, `*property-vector*`, `*version*`, and the rest of the
  globals initialised by `shen.initialise-environment` are populated correctly.
- The 41.1 KLambda sources (all 21 `.kl` files) are **vendored** under `klambda/`
  so a normal `git clone` is sufficient. `SHEN_KL_DIR` can still override the location.
- The full Shen test harness (`harness.shen`) loads, and `kerneltests.shen`
  runs end-to-end. **129 / 134 reports pass (96.27 % pass rate)** on the
  current 41.1 suite; see Counts below.
- Convenience test driver (`run-41.1-tests.lua`) reproduces the official run.

## Compiler changes required for 41.1

The 41.1 stlib and Prolog-compiled output stress LuaJIT's parser / VM limits
much harder than the old 22.4 kernel did. Several new code-generation
strategies were needed:

1. **Strict literal-data hoisting** (`try_lit_const`). Only `(cons L R)` trees
   are treated as compile-time literal data; other side-effecting calls
   (`set`, `lambda`, `shen.record-kl`, `protect`, …) are always evaluated.
   The previous heuristic over-hoisted, eliding the `(set *macros* …)` and
   most of the stlib's `(shen.record-kl …)` side effects.
2. **Flat cons-tree builder** (`MKTREE` + `compile_cons_tree`). The stlib's
   `(shen.record-kl name <source-tree>)` calls contain trees up to ~7 000
   cons cells / 216 levels deep. Naive nesting blows the Lua expression-
   complexity limit; the compiler now emits a flat blueprint array consumed
   by a runtime helper.
3. **Hoisted freeze closures** (`BIND` + per-defun `KB` table). Shen's Prolog
   compiler chains 60+ `(freeze …)` continuations inside argument positions.
   Each freeze body is now hoisted to a flat `KB[N] = function(...) … end`
   slot inside the enclosing defun; the use site emits `BIND(KB[N], cap…)`
   with no nested function literal, sidestepping LuaJIT's 65-syntax-level
   parser cap.
4. **Deep let-floating**. `(F1 a1 (F2 a2 (let X V B)))` is rewritten to
   `(let X V (F1 a1 (F2 a2 B)))` whenever the intermediate args are pure.
   Combined with (3) this flattens deeply nested CPS chains into a single
   block of `local` declarations.
5. **Right-spine call-chain flattening**. A long chain like
   `(shen.gc A (shen.gc A (shen.gc A …)))` in tail position is emitted as a
   sequence of `local` assignments rather than a single deeply-nested call
   expression (also a parser-cap fix).
6. **Self-evaluation in `eval`**. Vectors / streams / opaque tables handed
   to `eval-kl` by the macro expander are returned as-is, rather than
   compiled as KL forms.
7. **Port-specific primitives.** `shen.char-stoutput?` and `shen.char-stinput?`
   were missing — `writer.shen` and the reader expect every port to define
   them. Our streams are byte streams, so both return `false`.

## Known remaining failures

Five reports fail under the 41.1 official suite. They appear to involve
higher-order Prolog patterns (e.g. `(fn consit)` in `Prolog call`) and a
small number of typed-mode edge cases. Triage and a per-test breakdown
would be the natural next step; the harness keeps going past failures now
that `y-or-n?` is forced non-interactive in the test driver, so a quick
`grep "failed"` over the run captures them.

## How to Reproduce
```sh
# After a normal clone (kernel is vendored under klambda/)
luajit -e '
  local P = require("boot")
  P.load_kernel(false)
  P.initialise()
  print("version:", P.F["version"]())
'

# To run the official suite (set SHEN_TESTS_DIR or place tests at
# ../cl-source/ShenOSKernel-41.1/tests):
luajit run-41.1-tests.lua

# Microbenchmarks:
luajit bench.lua
```

You can still point at an external tree with `SHEN_KL_DIR=...` (useful when
working against a full ShenOSKernel checkout or a legacy 22.4 reference).
